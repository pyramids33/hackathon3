Demonstrate using the moneybutton to interact with api server 402 payment flow. This method takes a few swipes, so it should really use IMB, or moneybutton could implement this protocol as a single swipe on a '402' button type. This is not necessarily the best way to use it though, as you would normally purchase, store, and process these messages from multiple sources into some other format like a database or html before displaying to users. This is more about automated systems that collect data and sell it to each other. But there could be use cases that involve a Javascript apps automatically downloading and displaying data.
